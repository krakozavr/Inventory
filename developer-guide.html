Creating Compatible HTML Projects

Overview

This guide is for developers creating interactive HTML projects for StockProps. If you're a regular user, see the User Manual instead.

The Blob URL Challenge

StockProps loads your HTML in an isolated context called a "blob URL". This is necessary for security and offline functionality, but it creates constraints:

‚ö† Key Limitation: Your HTML runs in memory, not from the file system. Relative file paths don't work normally.

What This Means

Pattern Works? Notes
<img src="photo.jpg"> ‚úì Yes Auto-inlined as base64 during import
background-image: url('bg.jpg') ‚úì Yes Auto-inlined as base64
<link href="styles.css"> ‚úì Yes CSS inlined into <style> tag
<script src="app.js"> ‚úì Yes JS inlined into <script> tag
<script src="https://cdn.com/react.js"> ‚úì Yes External CDNs work normally
@font-face { src: url('font.woff2') } ‚úì Yes Fonts auto-inlined as base64
fetch('data.json') ‚úó No Local files not accessible
import X from './component.js' ‚úó No ES modules between local files fail

HTML Template Options

Option A: Inline Styles/Scripts (Simpler)

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Project</title>
    <style>
        /* All your CSS here */
        body { margin: 0; font-family: Arial, sans-serif; }
    </style>
</head>
<body>
    <img src="logo.png">

    <script>
        // All your JavaScript here
        console.log('App loaded');
    </script>
</body>
</html>

Option B: Separate Files (Better Organization)

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My Project</title>

    <!-- Local CSS - will be auto-inlined -->
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <img src="logo.png">

    <!-- Local JS - will be auto-inlined -->
    <script src="app.js"></script>
</body>
</html>
‚úì Both approaches work! Separate CSS/JS files are automatically inlined during import.

Best Practices

‚úì DO

  • Use separate CSS/JS files: <link href="styles.css">, <script src="app.js"> (auto-inlined)
  • OR use inline <style> and <script> tags
  • Use simple relative paths: src="image.jpg"
  • Put all files in same folder as HTML
  • Use external CDNs for libraries: https://unpkg.com/react
  • Embed data in JavaScript arrays instead of JSON files
  • Keep file names simple: 01.jpg, logo.png
  • Import all files together (HTML + CSS + JS + images)
  • Test by opening HTML directly in browser (should work offline)

‚úó DON'T

  • Use fetch() or XMLHttpRequest for local files
  • Use ES module imports between local files
  • Use complex folder structures: ./assets/images/photo.jpg
  • Include spaces or special characters in file names
  • Rely on server-side features (PHP, Node.js, etc.)

System Prompt:

Create an HTML project for StockProps, a mobile PWA that displays content in fullscreen mode.

Technical Constraints:
- Single HTML file with inline CSS and JavaScript (or separate files that will be auto-inlined)
- Images must use simple relative paths: src="01.jpg" (NOT src="./images/01.jpg")
- External libraries MUST use CDN links: https://unpkg.com, https://cdn.jsdelivr.net
- NO dynamic file loading (fetch/XMLHttpRequest for local files)
- NO ES module imports between local files
- Must work when loaded as blob URL in iframe sandbox

Supported Patterns:
‚úì <img src="photo.jpg"> (auto-inlined as base64)
‚úì background-image: url('bg.jpg') (auto-inlined)
‚úì <link rel="stylesheet" href="styles.css"> (auto-inlined to <style>)
‚úì <script src="app.js"></script> (auto-inlined)
‚úì @font-face { src: url('font.woff2') } (auto-inlined)
‚úì Custom fonts: WOFF, WOFF2, TTF, EOT, OTF
‚úì Inline <style> and <script> tags
‚úì <script src="https://unpkg.com/react"> (CDN)
‚úì <script src="tailwind.js"> (local Tailwind file)
‚úì Single-file React with Babel standalone
‚úì Data embedded in JavaScript (not separate JSON)

Unsupported Patterns:
‚úó fetch('local-file.json')
‚úó import X from './component.js'
‚úó Complex folder paths
‚úó Dynamic script loading from local files

Mobile Optimization:
- Viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1.0">
- Touch-friendly (large tap targets, swipe gestures)
- Mobile-first responsive design
- Interactive elements should NOT interfere with triple-tap exit gesture on edges

Output:
Complete, self-contained HTML that displays correctly on mobile devices in fullscreen.

Code Examples

Example 1: Image Carousel

<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        body {
            margin: 0; background: #000; color: #fff;
            display: flex; flex-direction: column;
            align-items: center; justify-content: center;
            height: 100vh; font-family: Arial;
        }
        img { max-width: 90%; max-height: 80vh; margin-bottom: 20px; }
        button { padding: 15px 30px; font-size: 18px; margin: 0 10px; }
    </style>
</head>
<body>
    <img id="viewer" src="01.jpg">
    <div>
        <button onclick="prev()">‚Üê Previous</button>
        <button onclick="next()">Next ‚Üí</button>
    </div>
    <script>
        const images = ['01.jpg', '02.jpg', '03.jpg', '04.jpg', '05.jpg'];
        let index = 0;

        function updateImage() {
            document.getElementById('viewer').src = images[index];
        }

        function next() {
            index = (index + 1) % images.length;
            updateImage();
        }

        function prev() {
            index = (index - 1 + images.length) % images.length;
            updateImage();
        }
    </script>
</body>
</html>

Testing Your Project

  1. Local test: Open HTML file directly in browser
    • Should work offline
    • Images should load
    • Interactions should work
  2. Mobile test: Copy folder to phone, import to StockProps
    • Check fullscreen display
    • Verify triple-tap exit works (use edge zones)
    • Test on actual device screen size

Common Issues

Images Don't Display

Cause: Complex paths or wrong folder structure

Fix: Use simple paths (src="image.jpg"), put images in same folder as HTML

CSS Not Loading

Cause: Trying to load CSS dynamically via JavaScript, or wrong file path

Fix: Use <link rel="stylesheet" href="styles.css"> in HTML (will be auto-inlined), or inline <style> tags. Don't use complex paths like ./assets/css/style.css - keep files in the same folder as HTML.

JavaScript Errors

Cause: Trying to fetch local files or use ES modules

Fix: Embed data in JavaScript, avoid fetch() for local resources

Advanced: Using React

For single-file React apps, use Babel standalone:

<!DOCTYPE html>
<html>
<head>
    <script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
    <script src="https://unpkg.com/react@18/umd/react.development.js"></script>
    <script src="https://unpkg.com/react-dom@18/umd/react-dom.development.js"></script>
    <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
    <div id="root"></div>
    <script type="text/babel">
        function App() {
            const [count, setCount] = React.useState(0);

            return (
                <div className="p-8 bg-gray-900 min-h-screen text-white">
                    <h1 className="text-3xl mb-4">Counter: {count}</h1>
                    <button
                        onClick={() => setCount(count + 1)}
                        className="bg-blue-500 px-6 py-3 rounded"
                    >
                        Increment
                    </button>
                </div>
            );
        }

        ReactDOM.render(<App />, document.getElementById('root'));
    </script>
</body>
</html>
